# Why agent-handoff?

AI coding assistants lose all context when a session ends. Starting a new session means re-explaining your entire codebase, the decisions you made, and what to do next. agent-handoff eliminates that friction.

✗ Without agent-handoff
Session 1: "Help me refactor auth"
Agent: Done! Added JWT support.
--- context lost ---

Session 2: "Continue auth refactor"
Agent: I don't have context
       about what you were
       working on. Can you
       re-explain the project?

Session 3: "..."
--- re-explaining again ---
✓ With agent-handoff
Session 1: "Help me refactor auth"
Agent: Done! Added JWT support.
/handoff auth-refactor
Saved. Tag: auth-refactor

Session 2:
/resume auth-refactor
+----------------------------+
| RESUMED: auth-refactor     |
| STATUS:  in_progress       |
| NEXT:    Add refresh token |
+----------------------------+
Agent: Continuing from where
       we left off...

# What you get

💾

Named snapshots

Tag any session for later. /handoff auth-refactor creates a uniquely named snapshot you can recall any time with /resume auth-refactor.

🧠

Persistent memory

Each handoff extracts stable facts into a memory layer — project conventions, your preferences, architectural decisions. Loaded automatically on every resume.

📄

Conversation transcript

The full conversation is saved as a transcript file. The agent uses it for verification when the summary isn't enough.

⚙️

Zero dependencies

Pure file writes, flat XML, works in any git repo. No database. No server. No install scripts. Copy two folders and you're done.

# Install

Clone the repo and copy the skill folders. Done.
